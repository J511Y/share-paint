# share-paint Development Team Operation (Fallback Mode)

서브에이전트 툴이 현재 게이트웨이 토큰 미스매치로 즉시 스폰 실패해서, 지금은 메인에서 운영계획 템플릿으로 대체 운영합니다.

## 팀 구성(운영 합의)
- PM: 범위/우선순위/스프린트 운영
- Backend: API/DB/인증/웹소켓
- Frontend: 화면/UX/상태관리
- Reviewer: 품질체크
- DevOps: 빌드/배포/모니터링

## 즉시 착수 제안(1주 스프린트)
1. **MVP 안정화**
   - 대결방 접속 안정성(재접속/세션 상태) 점검
   - 그림 업로드/피드 노출/좋아요/댓글 기본 동선 점검
2. **랜덤 주제/타이머 완결성**
   - 주제 풀 + 선택 API + 클라이언트 fallback
3. **모바일 UX 기본 점검**
   - 캔버스 조작 반응성, 화면 레이아웃, 버튼 터치 타깃

## 이번 주 스프린트 산출물(필수)
- 기능 정의서(최대 1페이지): MVP/비-MVP 분리
- API & 화면 변경 리스트(파일 경로 포함)
- 이슈/리스크 목록
- PR/체크포인트(테스트 포함)

## 일일 자동 리포트(요청 반영)
- 목표: 매일 오전 8시 `share-paint` 개선내역/신규 이슈/PR 목록/스프린트 범위 요약 발송
- 현재 상태: 게이트웨이 토큰 미스매치로 `cron.add` 실행 불가
- 해결 후 즉시 재등록 예정
